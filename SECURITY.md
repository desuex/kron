# Security Policy

## Supported Versions

Until v1.0, only the latest minor release is supported.

After v1.0, security fixes will be applied to the latest major version.

---

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report vulnerabilities privately via:

<kron@dsxm.org>

Include:

- Description of vulnerability
- Steps to reproduce
- Impact assessment
- Suggested mitigation (if known)

---

## Response Policy

- Acknowledge within 72 hours.
- Provide status update within 7 days.
- Coordinate responsible disclosure.
- Release patch and advisory when fixed.

---

## Scope

Security considerations for Kron are documented in `/docs/SECURITY.md`.

This includes:

- Execution isolation
- Privilege handling
- State integrity
- Determinism guarantees
- Supply chain practices

---

Kron prioritizes correctness and safety.
